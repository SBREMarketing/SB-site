# Sea Breeze — лендинг

Статический одностраничный сайт (без сборки) с поддержкой трёх языков: **RU / HE / EN**.
Язык отражается в маршруте — `/ru`, `/he`, `/en` — и сохраняется между визитами.

## Структура

```
.
├── index.html              # разметка (только содержимое, без стилей и логики)
├── styles/
│   └── main.css            # дизайн-система, компоненты, RTL, адаптивность
├── scripts/
│   ├── i18n.js             # движок локализации + языковая маршрутизация (ES-модуль)
│   └── main.js             # поведение интерфейса (ES-модуль, точка входа)
├── locales/
│   ├── ru.json             # переводы (русский — язык по умолчанию)
│   ├── he.json             # переводы (иврит, RTL)
│   └── en.json             # переводы (английский)
└── assets/
    └── images/             # изображения
```

## Локализация

Тексты хранятся **отдельно от разметки** — в `locales/<lang>.json`. В HTML переводимые
элементы помечаются атрибутами:

| Атрибут              | Что обновляется | Когда использовать                    |
| -------------------- | --------------- | ------------------------------------- |
| `data-i18n="key"`      | `textContent`     | обычный текст                         |
| `data-i18n-html="key"` | `innerHTML`       | если в значении есть разметка (`<br>`)  |

Значение в HTML служит запасным вариантом и применяется, пока не загрузится словарь.

### Как добавить строку перевода

1. Добавьте ключ во **все три** файла `locales/*.json`.
2. Повесьте `data-i18n="ваш.ключ"` на нужный элемент в `index.html`.

### Как добавить новый язык

1. Создайте `locales/<код>.json` со всеми ключами.
2. Добавьте код языка в `SUPPORTED_LANGS` в [`scripts/i18n.js`](scripts/i18n.js)
   (и в `RTL_LANGS`, если язык пишется справа налево).
3. Добавьте кнопку `<button data-lang="<код>">…</button>` в переключатель в `index.html`.

## Языковая маршрутизация

Выбор языка пишется в путь (`/ru`, `/he`, `/en`) через History API — без перезагрузки.
Источник языка при первой загрузке определяется по приоритету:
**сегмент пути → сохранённый выбор (`localStorage`) → язык браузера → `ru`**.
Поддерживаются кнопки браузера «назад/вперёд», а `?query` и `#hash` сохраняются.

### Важно: правило rewrite на сервере

Поскольку это одностраничный сайт, при **прямом заходе/обновлении** на `/ru` сервер
должен отдавать тот же `index.html`. Настройте rewrite на своём хостинге:

- **Netlify** — файл `_redirects`: `/*  /index.html  200`
- **Vercel** — `vercel.json`: `{ "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }] }`
- **nginx** — `location / { try_files $uri /index.html; }`
- **Apache** — `FallbackResource /index.html`

Также при деплое обновите домен в тегах `<link rel="alternate" hreflang…>` в `index.html`.

## Локальный запуск

ES-модули и загрузка JSON через `fetch` требуют HTTP — открыть `index.html` напрямую
через `file://` не получится. Поднимите любой статический сервер:

```bash
# Python
python -m http.server 8000

# Node
npx serve
```

Затем откройте `http://localhost:8000`.
